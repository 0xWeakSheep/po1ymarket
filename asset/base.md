# 新人快速入口

## 先知道你在做什么

你现在参与的是一个和 AI 配合的项目。

我们的目标不是单纯做一个网页，也不是单纯做一个接口，而是最终做到：

`你和 AI 对话 -> 告诉它一个盘口 -> 它自动调用我们的 API -> 帮你找这个盘口相关的信息 -> 再基于这些信息给你分析。`

所以你做的很多东西，尤其是 `skills`，本质上是在教 AI：

- 什么时候该调用我们的接口
- 怎么调用
- 返回结果怎么继续处理
- 最后怎么把分析结果说给用户

## 如何使用 AI

如果你是新人，先不要把 AI 当搜索框，更不要只让它“随便写点东西”。

你应该把 AI 当成一个协作者，用它来：

1. 帮你理解项目
2. 帮你拆任务
3. 帮你起草 skill
4. 帮你检查你写的内容
5. 帮你整理 commit 和 PR 说明

关于如何使用 Kimi Code，可以先看这个记录：

[Kimi Code 使用记录](https://www.kimi.com/share/19e0bf7e-f622-8042-8000-0000639732f7)

最简单的使用方式就是：

1. 告诉 AI 你现在要做什么
2. 告诉它你改的是哪个文件
3. 告诉它你希望输出什么结果
4. 让它先帮你整理，再帮你落文档或代码

## 如何快速理解什么是 skills

先看这个：

[什么是 skills](https://www.kimi.com/share/19e0bf95-4eb2-88b1-8000-0000447b0fc8)

你可以先用一句话理解：

`skill` 就是给 AI 的任务说明书。

它不是底层接口实现，而是告诉 AI：

- 什么时候触发
- 先做什么
- 后做什么
- 输出什么

在我们这个项目里，skill 的核心用途是：

- 让 AI 知道什么时候该去查盘口相关链接
- 让 AI 知道拿到链接后怎么继续读和分析

## 如何快速创建 skills

先看这个：

[skill-creator](https://skills.sh/anthropics/skills/skill-creator)

它可以帮助你更快搭出一个最小 skill 骨架。

你一开始不用追求复杂，先做一个最小版本：

1. 这个 skill 在什么场景下用
2. 它的输入是什么
3. 它调用什么
4. 它最后输出什么

如果你不知道从哪里开始，就先做“小 skill”，不要一开始做万能 skill。

例如：

- `fetch-market-sources`
- `triage-market-links`
- `summarize-market-evidence`

## 我们最终要的结果是什么

最终我们要的不是“写了几个 skill 文档”，而是一个真实可用的体验：

用户和 AI 说一句话，例如：

- “帮我看一下这个盘口”
- “分析一下这个 market_id”
- “这个问题今天值得关注什么信息”

然后 AI 能够：

1. 理解用户在问哪个盘口
2. 调用我们的 API
3. 获取相关链接和信息
4. 判断哪些信息更重要
5. 输出一段真正有用的分析

所以你做 skill 时，始终要记住：

`我们的目标不是让 AI 写得更像 AI，而是让 AI 真的会用我们的 API 去分析盘口。`
